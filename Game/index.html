Programming References

Java Type Rules

  • String values must be wrapped in double quotes: "hello"
  • int holds whole numbers only: 42, -5
  • boolean is only true or false (no quotes)
  • char uses single quotes: 'A'
  • double holds decimals without quotes: 3.14
  • Putting quotes around a number makes it a String, not a number
  • Putting quotes around true/false makes it a String, not a boolean

JavaScript Type Rules

  • String values are wrapped in quotes: "hello" or 'hello'
  • Number has no quotes: 42, -5, 0
  • Boolean is true or false without quotes
  • Even "false" with quotes is a String
  • Even "0" with quotes is a String
  • Zero 0 without quotes is a Number

Java vs JavaScript - Key Differences

  • Java is statically typed - you must declare types (String, int, boolean)
  • JavaScript is dynamically typed - types are determined by the value
  • In Java: String name = "Eppy"; (type on the left)
  • In JavaScript: let name = "Eppy"; (no type declaration needed)
  • Java uses "double quotes" for Strings, 'single quotes' for char
  • JavaScript can use either "double" or 'single' quotes for Strings

Common Mistakes to Avoid

  • Don't confuse true (boolean) with "true" (string)
  • Don't confuse 42 (number/int) with "42" (string)
  • Java int cannot hold decimals - use double instead
  • Java boolean cannot be 1 or 0 - only true or false
  • Java char needs single quotes, not double quotes
